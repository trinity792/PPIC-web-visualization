## What the Module Produces

The pipeline builds one file: `data/data-cleaned/housing-stress/HousingStress_Current.csv`. It measures **housing cost burden** ("housing stress") - the number and share of households that pay more than 30% or 50% of their income on housing - broken out by **location, year, race/ethnicity of householder, and tenure** (renter vs. owner). A verified live run against ACS vintage 2024 wrote 4,525 rows (50 states + 58 CA counties + 9 CA regions, times 9 race categories and 5 tenures, minus small-population suppressed strata).

Two things about the data are worth stating up front:

- **Each run adds one vintage on top of a seeded deep history.** The live pipeline fetches the single newest ACS 1-year vintage each time it runs and merges it into saved history. A dedicated backfill driver (`scripts/orchestrators/housing_stress_backfill.py`) seeds the deep-history file (`HousingStress_Historical.csv`) by walking every available vintage; the live pipeline then unions that seed with each new vintage, so deep years survive a rebuild of the current file. Note the Census **table-based Summary File** (this pipeline's source) only exists from **2022 onward** - the backfill covers 2022+, and the pre-2022 years (2012-2021) live in the legacy sequence-based format, so they are recovered from the set-aside legacy CSV rather than re-downloaded (a verified-safe bootstrap: legacy and V3 values match exactly on the overlap years). See "From Initial Refactor to Current State."
- **County and region numbers are approximate.** California counties and regions are built by aggregating PUMA-level estimates through a 2020 crosswalk. PUMAs do not nest inside county lines, so those figures are approximations; the state series is exact.

The refactor dissolved the legacy module's three near-identical geographic builders (`build_state_dataset`, `build_region_dataset`, `build_county_dataset`) - each of which re-declared its own download/clean/measure closures - into one shared code path, and it decoupled saving from charting (the legacy code only wrote the CSV as a side effect of rendering a chart). Charting moved entirely to the shared React frontend.

## Data Sources

One external source feeds the module:

| Source | Format | Geography | Years |
|---|---|---|---|
| U.S. Census Bureau **ACS 1-year table-based Summary File**, table **B25140** ("Tenure by Housing Costs as a Percentage of Household Income") plus its race iterations `B25140B`-`B25140I` | Two pipe-delimited files per (table, year): a `.dat` estimates file and a `Geos{year}1YR.txt` geography lookup, joined on `GEO_ID` | 50 US states, plus CA PUMAs aggregated to 58 counties and 9 regions | 2012 onward, **with a permanent gap at 2020** (ACS released no standard 1-year estimates that year) |

Nine table iterations are consumed. The base table (`b25140`) supplies the pre-aggregated "All" row; the eight suffixed tables supply the race categories. One deliberate choice is preserved from the legacy code: iteration **H** ("White alone, *not* Hispanic") is used for "White" instead of iteration **A** ("White alone", which includes Hispanic White), so that "White" and "Hispanic" do not double-count.

The source is acquired through a **three-step fallback cascade** (live download, then a manual CSV, then last-saved rows). The cascade returns a *discriminated payload* so each tier feeds the pipeline correctly: the live tier yields raw iteration frames that Phase 3 builds, while the manual and last-saved tiers yield already-cleaned contract rows that bypass the build and go straight to the merge (see "From Initial Refactor to Current State").

## The Dataset: Grain, Columns, and Geography

**Grain:** one row per `(Year, Geographic Level, Location, Race/Ethnicity, Tenure)`.

**Columns (in output order):**

| Column | Meaning |
|---|---|
| `Year` | Integer (2012+, never 2020) |
| `Geographic Level` | `State`, `Region` (9 CA regions), or `County` (58 CA counties) |
| `Location` | Two-letter USPS abbreviation for states, region name, or county name |
| `Race/Ethnicity` | One of 9: `All`, `White`, `Black`, `Asian`, `NHPI`, `AIAN`, `Multiracial`, `Hispanic`, `Other` |
| `Tenure` | `Total`, `Rented`, `Owned`, `Owned With Mortgage`, `Owned Without Mortgage` (renamed from the legacy `Label`) |
| `Number Over 30%` / `Number Over 50%` | Integer count of households paying >30% / >50% of income on housing |
| `Share Over 30%` / `Share Over 50%` | Proportion (0-1) of the tenure denominator; `NA` when the denominator is zero |

**Race reconciliation.** The 9 ACS iterations are reconciled toward the 7 canonical projections groups where the mapping is clean, plus two module-specific extras that have no projections counterpart: **"Other"** (some other race alone) and **"All"** (the base-table total). "All" is a *stored base row* from the un-suffixed table, **not** a sum of the race iterations - the iterations overlap and undercount by design, so summing them would not reproduce "All". Two semantic caveats are surfaced to the frontend: ACS "Multiracial" is two-or-more races of *any* ethnicity (projections defines it as non-Hispanic), and "Other" must never be folded into "Multiracial".

**Missing strata are absent, not imputed.** When ACS suppresses a race iteration for a small-population location-year, that `(Year, Location, Race)` group simply does not appear. Validation reports a missing *race* as a warning (suppression is expected) but a missing *tenure* for a race that is present as an error (the cost-burden math should always yield all five tenures).

## How the Pipeline Runs: Five Phases

The orchestrator `build_housing_stress_dataset()` runs five phases. Each is wrapped so any exception re-raises as a `HousingStressPipelinePhaseError` tagged with the phase name.

### Phase 1 - Setup and Load

Resolve the three config dicts (`paths`, `sources`, `schemas`) and the shared California geography, then load two history sources - the current contract CSV and the immutable deep-history seed - and union them (preferring the current output row-for-row) into one `historical` frame.

### Phase 2 - Acquisition

First, **resolve the latest vintage**: probe backward from the current calendar year (skipping 2020 without spending a probe, retrying a transient network fault before giving up) until the base `b25140` table for California downloads and parses. Then acquire the whole vintage with a single fallback:

- **Live tier:** download each of the 9 iteration national `.dat` files **once** (through an on-disk cache) and slice both scopes from the same frames - the state-summary rows (`GEO_ID` prefix `0400000US`) and the California rows (whose PUMA rows feed the county and region rollups).
- **Fallback tiers:** if the live download fails, fall back to a manual contract file, then to the last-saved rows.

The acquisition returns a discriminated payload plus the `source_failed` / `source_used_manual` flags that the run record surfaces.

### Phase 3 - Build Levels

`build_all_levels` runs the one shared code path three times, differing only in the geography step:

- **State:** filter to the 50 state abbreviations (DC and PR excluded), use the USPS abbreviation as `Location`.
- **Region / County:** extract PUMA ids from `GEO_ID`, aggregate estimates to the target geography via the 2020 crosswalk, map region ids 1-9 to names.

After the geography step, all three apply the identical transforms: normalize columns, compute the tenure cost-burden measures, reconcile race labels, and stamp the year and geographic level. The three level-frames are concatenated and sorted, then run through `validate_cleaning_output` before the merge. (When acquisition fell back to a contract-shaped payload, the build is skipped entirely and those already-cleaned rows go straight to Phase 4.)

### Phase 4 - Merge

Validate the incoming vintage's stratification completeness, then atomically replace any overlapping historical year (drop every historical row whose year is in the incoming frame, append the incoming rows whole - never a key-level upsert). The merge base is the seed+current union so deep years are retained, but the change flag is measured against the **current output file** (not the union), so the first run after a fresh seed materializes the full series and later runs no-op until a genuinely new vintage lands.

### Phase 5 - Finalize and Save

Enforce contract column order and types, run final validation (required columns, row-count bounds, all three levels present, year range and excluded-year checks, non-negative counts, shares in [0, 1], no duplicate keys), and - only if new data was detected - archive the prior file with a timestamp and write the new CSV **atomically** (staged temp file + `os.replace`), so a crash mid-write can never truncate the file that also serves as the next run's baseline.

## Key Calculations

**The cost-burden measures.** For each of five tenures, four measures are derived from the B25140 estimate columns `E001`-`E013`. The column semantics (from the B25140 data dictionary) are configured once in `schemas.py` rather than hard-coded three times as in the legacy module:

| Tenure | Number Over 30% | Number Over 50% | Denominator (for Share) |
|---|---|---|---|
| Total | `E003 + E007 + E011` | `E004 + E008 + E012` | `E001` |
| Rented | `E011` | `E012` | `E010` |
| Owned | `E003 + E007` | `E004 + E008` | `E002 + E006` |
| Owned With Mortgage | `E003` | `E004` | `E002` |
| Owned Without Mortgage | `E007` | `E008` | `E006` |

`Share Over 30%` = `Number Over 30% / Denominator` (and likewise for 50%). A **zero denominator yields `NA`, not infinity** - division-by-zero is handled explicitly (`np.errstate` + a `.where(denominator != 0)` mask) instead of being suppressed by the legacy global `warnings.filterwarnings("ignore")`.

**PUMA aggregation (the approximate step).** County and region estimates are built by inner-joining PUMA rows to the 2020 crosswalk on `PUMA_ID` (unmatched PUMAs drop, matching legacy behavior) and summing the estimate columns by target geography *before* the tenure math runs. Because a PUMA straddling two counties is assigned wholesale to one, county and region figures are approximations.

**Vintage resolution.** Unlike the legacy bare-`except` probe that decremented the year on *any* error, `resolve_latest_vintage` distinguishes "not published" (a 404) from a transient network fault (which it retries a few times before giving up, so a blip does not silently resolve an older vintage) from "published but malformed" (a parse `ValueError`, which propagates). It skips the excluded 2020 without spending a probe and walks down to `earliest_year` if Census is late, rather than failing on a fixed 3-year window.

## Architecture and Boundaries

The module follows the standard three-layer V3 shape:

- **Shared layer** (`scripts/shared/`) - HTTP downloads, California geography (canonical county/region names + region ordering), generic DataFrame validators. Never imports anything housing-stress-specific.
- **Domain layer** (`scripts/housing_stress/`) - config, acquisition, cleaning, geography, aggregation, merging, validation, and output.
- **Orchestrator** (`scripts/orchestrators/housing_stress_pipeline.py`) - wires the domain modules into the five phases and owns the fallback logic.

The dependency direction is strict: shared -> housing_stress -> orchestrator. There is **no Python visualization layer** - the legacy `visualize_line/bar()` methods (which also triggered the save as a side effect) were dropped in favor of the shared React charting UI. The frontend triad (`lib/visualization/moduleSchemas/housingStress.js`, `lib/data/housing_stress.js`, `app/api/housing-stress/route.js`) only ever *reads* the contract CSV and is described at the interface level in the second half.

## Phase 2: Acquisition

### `scripts/housing_stress/acquisition/acs_sf_downloader.py`

- **`ACSTableUnavailableError`** - signals "not published" (404 / missing file), distinct from a parse failure.
- **`_is_missing_file_error(error)`** - tests `error.status_code == 404` (the structured code now carried by `HTTPDownloadError`), falling back to a message substring only when no status is present, so a timeout (no status) reads as transient rather than as a missing file.
- **`_read_pipe_delimited(content)`** - `pd.read_csv(io.BytesIO(content), sep="|", index_col="GEO_ID")`.
- **`_fetch_content(url, headers, timeout, cache_dir, cache_max_age_days)`** - fetches a URL's bytes, honoring an on-disk cache keyed by the URL basename: a cached file younger than the window is returned without a network request, otherwise the file is fetched and written to the cache atomically.
- **`download_national_table(..., cache_dir=None)`** - fetches the national `.dat` and the `Geos` lookup (through `_fetch_content`), joins them on `GEO_ID`, and returns **all** rows. Raises `ACSTableUnavailableError` on a missing file (so the caller can step back a year) but `ValueError` on a present-but-malformed file (missing geo columns). *This is the key dedup* - it downloads each table once nationally, and callers filter it in memory.
- **`download_all_national_tables(year, source_settings, headers, timeout, cache_dir=None)`** - the single-fetch entry point: downloads each of the 9 iteration national tables exactly once (through the cache), from which the orchestrator slices both the state and CA scopes. Records a suppressed non-base iteration in `missing_iterations`; raises if the base table is missing.
- **`get_acs_table(...)`** / **`download_all_iterations(...)`** - thin wrappers that keep one `STUSAB`; both key the "is base table" test on `tblid == base_table_id` (not a positional index). *Derived from* legacy `get_data()` and the per-iteration loops.

### `scripts/housing_stress/acquisition/source_fallback.py`

- **`resolve_latest_vintage(state, source_settings, headers, timeout, max_year_lookback, excluded_years, earliest_year=None, cache_dir=None, probe_retry_attempts=1)`** - probes backward from `datetime.now().year`, skipping `excluded_years` **without spending a probe**, walking down to `earliest_year`, and accepting the first year whose base table downloads and parses. `_probe_base_table` retries a transient `HTTPDownloadError` `probe_retry_attempts` times before treating it as "not published"; a 404-style `ACSTableUnavailableError` advances immediately and a parse `ValueError` propagates. *Derived from* the legacy bare-`except` year-decrement loop.
- **`acquire_with_fallback(live_download_fn, manual_fn, saved_rows_fn)`** - tries the live download, then `manual_fn()` (a contract frame or `None`), then `saved_rows_fn()`, returning a discriminated `AcquisitionResult(kind, data, source_failed, used_manual)` where `kind` is `ITERATION_FRAMES` (live; Phase 3 builds it) or `CONTRACT_ROWS` (manual/last-saved; Phase 3 bypasses the build). *Derived from* the legacy inline fallback.

## Phase 3: Cleaning and Geographic Aggregation

### `scripts/housing_stress/cleaning/column_normalization.py`

- **`strip_table_prefix(df)`** - strips the `B25140xxx_` prefix via `str.replace(r"^.*_", "")`, then **validates** that each expected `E001`-`E013` column is present exactly once (raises otherwise) - hardening the fragile legacy regex.
- **`drop_margin_of_error_columns(df)`** - drops columns matching exactly `M\d{3}` (keeps `MARGIN_NOTE` etc.).

*Legacy lineage:* replaces the inline `df.columns.str.replace(...)` prefix strip and MOE drop.

### `scripts/housing_stress/cleaning/cost_burden_measures.py`

- **`compute_tenure_measures(df, id_columns, schema_config)`** - the single shared implementation of the tenure math (the biggest dedup in the module). For each of the five tenures it sums the configured numerator/denominator columns, divides for the shares under `np.errstate(divide="ignore", invalid="ignore")`, masks zero-denominator shares to `NA`, and stacks one block per tenure. Raises if any formula-referenced estimate column is absent. Uses `numpy` + `pandas`. *Derived from* the tenure/burden math copy-pasted into all three legacy builders.

### `scripts/housing_stress/cleaning/race_ethnicity_mapping.py`

- **`get_canonical_race_groups()`** and **`reconcile_race_label(df, race_column, reconciliation_map)`** (maps raw iteration labels to canonical, raising on any unmapped label). Owns `CANONICAL_RACE_GROUPS` (9) and `RACE_ITERATION_MAP`. *Derived from* the legacy `table_id_to_race_dict` and its iteration loop.

### `scripts/housing_stress/geography/puma_aggregation.py`

- **`extract_puma_id(df)`** - keeps rows by the structural `GEO_ID` PUMA summary-level prefix `795` (e.g. `795P200US0600101`; not the `NAME` text, so a label-wording change cannot silently drop them, and the bare sumlevel is stable across vintages whose middle segment differs) and parses the trailing 5 digits into an integer `PUMA_ID` (handling `GEO_ID` as index or column). Raises if no PUMA rows are found or an id is unparseable.
- **`aggregate_pumas_to_geography(df, crosswalk_path, crosswalk_geo_column, estimate_columns, output_location_column)`** - validates that the crosswalk has the `pumace` and geography columns (a clear file-naming error, not an opaque `KeyError`), inner-joins on `PUMA_ID` (unmatched PUMAs drop, with a warning on a large drop), renames the target column, and sums the estimate columns by geography. The approximate step.
- **`map_region_ids_to_names(df, region_column, region_id_to_name)`** - replaces numeric region ids 1-9 with canonical names from the explicit `region_id_to_name` map; raises on an unknown id.

*Legacy lineage:* replaces the legacy PUMA-id extraction, crosswalk merge, and group-by-sum inside `build_county_dataset` / `build_region_dataset`.

### `scripts/housing_stress/aggregation/geographic_levels.py`

- **`build_state_rows` / `build_region_rows` / `build_county_rows`** - the three builders, each iterating the iteration frames, applying the geography step, then the shared `_measure_and_tag` (reconcile race, compute tenure measures, stamp year and level). Region/county builders call the PUMA helpers; the state builder filters to the 50 abbreviations.
- **`build_all_levels(ca_frames, state_frames, year, paths, schema_config, geography)`** - runs all three and concatenates + sorts into one vintage frame.
- Helpers: `_contract_columns`, `_measure_and_tag`, `_prepared_estimates` (strip + drop MOE), `_region_id_to_name` (returns the explicit `schema_config["region_id_to_name"]` literal).

*Legacy lineage:* replaces the three duplicated closures `build_state_dataset` / `build_region_dataset` / `build_county_dataset`, collapsing them to one code path that differs only in the geography step.

## Phase 5: Validation and Output

### `scripts/housing_stress/validation/housing_stress_validators.py`

- **`validate_cleaning_output(df, schema_config)`** - a post-build validator (required columns, null critical columns, non-negative counts, shares in [0, 1], canonical tenure/race), now **wired into Phase 3** so a malformed live build is caught before the merge.
- **`validate_stratification_completeness(df, schema_config)`** - the completeness gate used during merge: per `(Geographic Level, Location, Year)` group, a missing *race* is a WARNING (suppression expected) but a present race missing any *tenure* is an ERROR. Only errors fail the vintage.
- **`validate_housing_stress_dataset(df, validation_config)`** - the final gate: required columns, row-count bounds, all three levels present, year range, excluded-year (2020) check, non-negative counts (now reading the same `nonnegative_columns` key the config supplies, so the check actually runs), shares in [0, 1], and duplicate-key detection. Composes the shared validators.

### `scripts/housing_stress/output/finalize_dataset.py`

- **`prepare_output(df, schema_config)`** - checks contract columns, casts `Year` to int and the measures to numeric/float, orders columns, and sorts. Raises on a missing contract column.
- **`archive_and_save(df, current_path, archive_directory)`** - compares the new CSV against the existing file by streamed SHA-256; on a match returns `None` (no write, file left byte- and mtime-identical), otherwise copies the old file into the archive with an `mm-dd-yy` stamp and writes the new bytes **atomically** (staged `.tmp` + `os.replace`). *Replaces* the legacy pattern where saving was a side effect of chart rendering.

### `scripts/orchestrators/housing_stress_pipeline.py`

- **`HousingStressPipelinePhaseError` / `_raise_phase_error`** - tag failures with their phase.
- **`_acquire_live_frames(...)`** - downloads the 9 national tables once (through the cache) and slices both the state-summary rows and the CA rows from the same frames - the single-fetch fix that turned ~19 national-file fetches per run into ~9.
- **`_manual_contract_frame(...)` / `_contract_rows_from_payload(...)`** - read the manual fallback file(s) as one contract frame (raising a clear error if not contract-shaped) and validate a fallback payload's shape before it bypasses the build.
- **`build_housing_stress_dataset(config=None, logger=None)`** - runs the five phases (threading a logger through resolution, acquisition, and merge) and returns `{dataset, new_data, source_failed, source_used_manual, resolved_year, output_path, row_count}`.
- **CLI:** `python -m scripts.orchestrators.housing_stress_pipeline` (run as a module from the repo root, so the absolute `scripts.` imports resolve) prints the vintage, row count, and write status.

*Legacy lineage:* replaces `build_full_dataset()` and its three nested builder closures.

### `scripts/orchestrators/housing_stress_backfill.py`

- **`backfill_housing_stress_history(config=None, logger=None, start_year=None, end_year=None, acquire_frames_fn=None, include_legacy=True)`** - the one-time deep-history seed driver: reuses the live acquisition + build + merge path with the year pinned, walking every vintage from `earliest_year` to the latest available (skipping 2020 and any year the table-based Summary File does not carry, i.e. pre-2022). It then bootstraps the pre-2022 years from the legacy CSV via **`_load_legacy_seed`** - renaming the two legacy columns (`Race/ethnicity`, `Label`), reconciling the raw race labels with the live cleaner's map, and taking the years below the earliest V3 year (excluding 2020). Those legacy years are unioned with a permissive completeness gate (they legitimately carry small-population suppressed tenure strata, which the final validator - not the per-vintage completeness gate - is the right place to accept), and the accumulated 2012-2024 series is written to the immutable `HousingStress_Historical.csv`. The existing seed is archived before the new one is written, so a rebuild never destroys the prior good seed. Runnable on demand.

## From Initial Refactor to Current State

The refactor first split the legacy `housingstress_code.py` monolith into the phase modules described above, dissolving the three near-identical geographic builders into one shared code path. A later reliability, robustness, and efficiency pass then hardened several parts of that initial refactored state into what this guide documents. This section records those changes: what the first-cut refactor left in place, and what it became, so a reader understands why the current code is shaped the way it is.

### The fallback cascade became a real degrade path

The first-cut refactor advertised a three-tier acquisition cascade - live download, then a manual CSV, then last-saved rows - but only the live tier actually worked. `acquire_with_fallback` returned whatever each tier produced, and Phase 3 treated that payload as a **dict of iteration frames**, iterating `.items()` and running each value through the estimate-column transforms. The live tier does return iteration frames, but the manual tier returned a single contract-shaped DataFrame (`pd.read_csv`) and the last-saved tier returned already-cleaned historical rows. Iterating a DataFrame's `.items()` yields `(column, Series)` pairs, so a fallback fed `strip_table_prefix` a `Series` and crashed with an opaque `AttributeError` - the entire fallback cascade was decorative, and a live-acquisition failure took the whole run down rather than degrading.

Acquisition now returns a **discriminated payload**: `acquire_with_fallback` yields an `AcquisitionResult(kind, data, source_failed, used_manual)` where `kind` is `ITERATION_FRAMES` (live) or `CONTRACT_ROWS` (manual / last-saved). Phase 3 branches on it - it builds the iteration frames as before, or, for a contract-shaped payload, **skips the build entirely** and passes the already-cleaned rows straight to the merge. Per the maintainer's decision, the manual file holds finished contract rows (validated with a clear shape check that raises instead of crashing the builder), and a live failure with history present degrades to the last-saved rows, surfacing `source_failed` in the run record rather than failing loudly. An orchestrator-level test now drives the pipeline with live acquisition forced to fail and asserts a degraded-but-valid run instead of a crash. The same restructure unified the two former per-scope acquisition passes into one, which also enabled the single-fetch efficiency fix below.

### Deep history became a separate, seeded artifact with a backfill driver

The first-cut refactor pointed `historical_data_path` at the pipeline's own output (both were `HousingStress_Current.csv`) and fetched only the newest vintage per run, so deep history accumulated only by running repeatedly over time - and a from-scratch build produced a **single-year** dataset, not the "2012 through latest" series the contract describes. That self-perpetuating history also carried a slow-corruption risk: the completeness gate validates only the *incoming* vintage, so a corrupt historical year written by a prior bad run was never re-checked.

Deep history now lives in its own immutable file, `HousingStress_Historical.csv` (`historical_data_path`, decoupled from `current_data_path`), read-only to the live pipeline and produced by a new one-time driver, `scripts/orchestrators/housing_stress_backfill.py`. That driver reuses the live acquisition + build + merge path with the year pinned rather than resolved, walking every vintage from `earliest_year` to the latest available and writing the accumulated series to the seed. The first real backfill run surfaced a source limitation: the Census table-based Summary File only carries **2022 onward** (2018/2019/2021 return 404), so the driver builds 2022+ live and bootstraps the pre-2022 years from the set-aside legacy CSV - which turned out to hold the same values V3 computes (byte-identical on the 2022/2023 overlap), needing only the two column renames and the race-label reconciliation the config already defines. The legacy years carry small-population suppressed tenure strata, so they are unioned with a permissive completeness gate and accepted by the final validator rather than the per-vintage gate (which exists to catch build bugs in fresh vintages). The result is a full 2012-2024 seed. Phase 1 loads both the current output and this seed and unions them with `combine_history_sources`, preferring the current output row-for-row while the seed still supplies any deep year the current file may have lost; `load_historical_baseline` and `load_canonical_dataset` both return an empty contract frame (with a loud warning for a missing seed) rather than crashing, so the module cold-starts on live data alone. Per the maintainer's decision the seed is backfilled once and committed, with the driver runnable on demand to rebuild it; because writing the seed goes through `archive_and_save`, a rebuild archives the prior seed before overwriting, so a failed rebuild leaves the previous good copy recoverable. A frontend "rebuild on demand" control remains an operational follow-up (it needs a running server and hits live Census).

### Vintage resolution gained retries and an adaptive window

`resolve_latest_vintage` advanced past a transient `HTTPDownloadError` as well as a 404-style `ACSTableUnavailableError` - deliberate, since the Census host hangs rather than 404s for some unpublished vintages, but it meant a transient timeout on a genuinely-published newest vintage silently resolved the **previous** year. And `max_year_lookback = 3` probed only the current year and two prior (minus the excluded 2020, which consumed a slot), so a late Census release beyond that window failed Phase 2 outright.

Resolution is now both retrying and adaptive. `_probe_base_table` retries a transient fault `probe_retry_attempts` (3) times before treating it as "not published," so a blip recovers rather than resolving an older year, while a real 404 still advances immediately and a parse `ValueError` still propagates. The walk skips excluded years **without spending a probe** and descends to `earliest_year` (2012) if Census is late, bounded by a wider `max_year_lookback` (6) that guards against a wholesale outage rather than a merely-late release. The resolution probe also shares the download cache, so the base table fetched to resolve the year is reused rather than re-downloaded.

### The validators became real acceptance gates

Two validators were effectively inert. `validate_housing_stress_dataset` iterated `validation_config.get("number_columns", [])` for its negative-value check, but the config supplies the count columns under `nonnegative_columns` - so the final-stage non-negative check was a silent no-op. And `validate_cleaning_output`, the natural post-build gate, had no caller at all (the same "tested but unused" smell flagged in the sibling modules). The final validator now reads `nonnegative_columns` (so a negative count actually fails the save), and `validate_cleaning_output` is wired into Phase 3 immediately after the build, catching a malformed live vintage before it reaches the merge. Both have new tests with crafted violations.

### National tables are fetched once and cached

`sources.py` advertised `cache_max_age_days = 30`, but nothing honored it, and the same national files were pulled repeatedly within a run: the resolution probe fetched the base table, the state pass fetched all 9 national `.dat` files, and the CA pass fetched all 9 again - roughly 19 national-file fetches (double that counting the paired geo files) for data that could be fetched once. `download_national_table` now reads from and writes to an on-disk cache keyed by the URL basename (younger than the window is served without a request), and the orchestrator's `_acquire_live_frames` downloads each iteration's national table exactly once through `download_all_national_tables`, slicing both the state-summary rows and the CA rows from the same frames. A run now issues ~9 iteration fetches instead of ~19, and the backfill driver inherits the cache so re-runs are cheap. A test pins the per-run `fetch_response` count.

### The contract file is written atomically

`archive_and_save` overwrote the output in place with `write_text` - so a process killed mid-write left a truncated CSV that, because the contract file also seeded the baseline, became the next run's silently-shortened history. It now compares by streamed SHA-256 (leaving an unchanged file byte- and mtime-identical) and writes the new bytes to a staged `.tmp` sibling, then `os.replace`s it into place; ordering is archive old -> write tmp -> atomic replace, matching Population and Housing and Components. The immutable seed closes the remaining exposure, since a bad current write can no longer poison the deep-history baseline.

### Region names, iteration maps, and error classification stopped depending on order and message text

Three latent coupling points were removed. The region-id -> name map was built by `enumerate(regions_mapping)`, coupling the crosswalk's numeric ids to the iteration order of a dict shared across three modules - a reorder there would have silently relabeled every California region. It is now an explicit literal keyed by the crosswalk's actual ids, with an import-time assertion that its names match the shared California region set (so a rename fails loudly without depending on ordering). The 9-iteration table map was declared twice (raw labels in `sources.py`, canonical labels in `schemas.py`); both now derive from a single owner, `config/table_iterations.py`, which also names `BASE_TABLE_ID` explicitly and asserts the base table is first, so the "is this the base table" check keys on the id rather than a positional index. And `HTTPDownloadError` now carries a structured `status_code`, so `_is_missing_file_error` tests `error.status_code == 404` instead of sniffing the message - a shared-layer improvement every module's downloader inherits.

### PUMA detection and change detection stopped being text-fragile

`extract_puma_id` detected PUMA rows by `NAME.str.contains("PUMA")`, so a change to ACS label wording would silently drop every PUMA row; it now detects them by the structural `GEO_ID` summary-level prefix `795` (the invariant across vintages, verified against live 2024 data - the full segment is `795P200US…`) and raises if none are found. `aggregate_pumas_to_geography` hard-coded the crosswalk headers and would raise an opaque `KeyError` on a rename; it now validates the header on load with a clear, file-naming error and warns on a large unmatched-PUMA drop. And `detect_new_data` compared frames across a CSV round-trip with `DataFrame.equals`, so an int-vs-float dtype drift - and, as the first live run exposed, the ~1e-16 float ULP noise between a freshly-computed share and the same value round-tripped through the CSV - reported spurious "new data"; it now compares numeric columns with an absolute tolerance (`np.allclose`, `atol=1e-9`) and non-numeric columns exactly, so the change flag agrees with the serialized-bytes decision the write step makes and no longer disagrees with a byte-identical no-op write.

### Structured logging gained phase coverage

The shared logging layer was already implemented and wired through `execute_pipeline_run` before this pass, but only Phase 5 emitted a record, and the summary's `used_manual` key did not match `run_records`' `FALLBACK_FLAG_PATTERN`, so a manual-fallback run was not marked `recovered`. The orchestrator now logs at acquisition (which tier fired and the resolved year) and merge as well as the Phase 5 quality check, and the summary key was renamed to `source_used_manual` so a manual run is correctly flagged as recovered.
